REQUIREMENTS

1) Cloudinary Asset Library
Implement an ingestion utility that:
- Takes an array of Cloudinary Collection share URLs
- Extracts all video assets from each collection
- Normalises them into a single merged asset library with:
  - assetId (stable unique)
  - cloudinaryUrl
  - sourceCollectionUrl
  - type = "video"
  - posterUrl (generate thumbnail if possible)
  - duration/width/height if available
- Caches results locally (file or local storage) to avoid slow reloads
- Provides a manual refresh button in /admin/assets

2) Data Model
Create local data storage for case studies, either:
- /src/data/caseStudies.ts (preferred) or JSON
Fields:
- id
- title
- clientOrProjectName
- category
- summary
- theBrief
- approach (array)
- outcome
- services (array)
- featuredVideo (assetId reference)
- supportingVideos (array of assetId references)
- hero (boolean)
- tags (array)
- location (optional)
- year (optional)

3) Public Pages
- /case-studies (index):
  - Featured hero studies at top (3 items)
  - Grid of remaining
  - Filter by category and tags
  - Each card shows poster thumbnail and quick summary
- /case-studies/:id (detail):
  - Above fold: title, summary, embedded featured video
  - Sections: Brief, Approach, Outcome, Services, Supporting videos
  - CTA button: Enquire (links to contact form)

4) Showreel Handling (special case)
The showreel is a pillar asset, not a normal case study.
Implement:
- A homepage component/slot for the Showreel embed and a strong "Watch Showreel" CTA
- Also allow a Showreel page, e.g. /showreel (optional) or feature it at top of /case-studies as an intro block
- Do not bury it inside the normal case studies grid as if it were a client project

5) Admin-lite UI (no nav link)
- /admin/assets:
  - List all ingested assets with poster previews
  - Filter by sourceCollectionUrl
  - "Refresh assets" button
- /admin/case-studies:
  - Create/edit case studies
  - Pick featured video from asset library (assetId)
  - Add supporting videos, reorder them
  - Toggle hero
  - Preview card and detail layout

If the site already has auth, protect these routes. Otherwise keep it hidden and simple.
