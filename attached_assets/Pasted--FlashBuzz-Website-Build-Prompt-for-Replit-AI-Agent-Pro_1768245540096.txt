# FlashBuzz Website Build Prompt for Replit AI Agent

## Project Overview

Build a premium video production portfolio website for **FlashBuzz** (one word, lowercase ‘b’) — a filmmaker-led video production company based in Banbury, Oxfordshire, UK.

**Critical Brand Rule**: The brand name is “FlashBuzz” with a capital F and lowercase b. Never write “FlashBuzz” with capital B.

-----

## Core Positioning & Brand Identity

**Primary Positioning Statement**: “Documentary craft for business stories that matter.”

**Brand Essence**: FlashBuzz brings a filmmaker’s discipline—story architecture, authentic interview technique, and cinematic execution—to organisations where generic corporate video fails. This is video production powered by an award-winning filmmaker (23 international awards, 9.2 IMDb rating for “Time Spent”), not a content mill or generic agency.

**Tone of Voice**:

- Calm authority — knowledgeable, confident, trustworthy without corporate stiffness
- Human over corporate — conversational language demonstrating expertise through insights, not jargon
- Confident understatement — letting quality speak rather than over-claiming
- Premium but accessible — avoiding urgency-driven language or pushy sales tactics

**Voice Examples**:

- ✅ “We make films that change how people feel about your business.”
- ✅ “Before we ever press record, we understand who you are and why your story matters.”
- ❌ “We create award-winning cinematic content” (too generic/overused)
- ❌ “Full-service production capabilities” (too corporate/jargon-heavy)

-----

## Technical Stack & Architecture

**Framework**: Next.js 14+ with React, App Router
**Styling**: Tailwind CSS for utility-first responsive design
**Video Hosting**: Vimeo embeds (use Vimeo Player API for custom controls if needed)
**Blog/Journal**: MDX or markdown-based with static generation
**Forms**: Contact form with validation (consider Formspree, Web3Forms, or similar service)
**Performance**: Optimise for Core Web Vitals — fast loading, minimal CLS
**SEO**: Next.js metadata API, structured data (JSON-LD for LocalBusiness, VideoObject)
**Analytics**: Prepare for Google Analytics 4 integration (can be added post-launch)

## Design & Visual Identity Guidelines

**Colour Palette**:
Suggest a premium, cinematic palette:

- Primary: Deep charcoal or black (#1a1a1a) for authority
- Secondary: Warm accent (e.g., warm bronze #B8956A or cinematic teal #2D5F6F)
- Neutrals: Light grey (#f5f5f5) backgrounds, white (#ffffff)
- Use colour sparingly for CTAs and accents

**Typography**:

- Headings: Modern sans-serif with authority (e.g., Inter, Manrope, Space Grotesk)
- Body: Highly readable sans-serif (e.g., Inter, Open Sans, System fonts)
- Consider serif for accent/pull quotes for editorial feel (e.g., Merriweather)
- Hierarchy: Clear size differentiation between H1 > H2 > H3 > body

**Imagery & Video**:

- All imagery should feel premium, authentic, not stock-photo generic
- Use behind-the-scenes shots, production stills, location photography
- Vimeo embeds should have custom thumbnails (not auto-generated)
- Ensure video embeds are responsive and don’t break layout on mobile

**Spacing & Layout**:

- Generous white space — never cramped
- Consistent padding/margins (use Tailwind spacing scale)
- Sections should breathe
- Grid systems for consistency

**Interactive Elements**:

- Subtle hover effects on buttons (slight scale or colour shift)
- Smooth transitions (0.3s ease)
- Card hover states reveal more information
- Video thumbnails: play icon overlay on hover

**Mobile-First Design**:

- All sections must be fully responsive
- Navigation transforms to hamburger menu on mobile
- Video embeds must maintain aspect ratio
- Forms must be easy to complete on mobile

## SEO & Technical Requirements

**On-Page SEO**:

- Unique meta titles and descriptions for every page
- Proper heading hierarchy (one H1 per page, logical H2-H6 structure)
- Alt text for all images (descriptive, includes keywords naturally)
- Internal linking between related pages (services ↔ sectors ↔ case studies)
- Breadcrumb navigation on deeper pages

**Structured Data (JSON-LD)**:
Implement for:

- LocalBusiness schema (homepage, contact page)
  - Include: name, address, phone, geo coordinates, service areas
- VideoObject schema (case study pages, portfolio)
  - Include: name, description, thumbnailUrl, uploadDate, duration
- BreadcrumbList schema (all interior pages)
- FAQPage schema (FAQ sections)

**Technical SEO**:

- Semantic HTML5 (proper use of `<header>`, `<nav>`, `<main>`, `<article>`, `<footer>`)
- Canonical URLs
- XML sitemap (auto-generated via Next.js)
- Robots.txt
- Fast loading times (aim for <3s LCP)
- Optimise images (next/image for automatic optimisation)
- Lazy loading for videos and below-fold content

**URL Structure**:

- Clean, descriptive URLs
- Lowercase with hyphens
- Examples:
  - `/work/recruitment-video-oxfordshire-logistics-firm`
  - `/services/brand-corporate-films`
  - `/sectors/financial-services`
  - `/video-production-banbury`

**Accessibility**:

- WCAG 2.1 AA compliance
- Keyboard navigation support
- Screen reader friendly
- Sufficient colour contrast ratios
- Skip to content link
- Focus indicators on interactive elements

## Content Guidelines for AI Agent

## Vimeo Integration Notes

**Vimeo Player API**:

- Use `@vimeo/player` npm package for advanced controls
- Implement responsive embed (16:9 aspect ratio maintained)
- Options to consider:
  - Autoplay (muted) for homepage hero
  - Custom play button overlays
  - Disable Vimeo logo/controls for cleaner look (requires Vimeo Plus/Pro)
  - Track video engagement if analytics integrated

**Example Vimeo Embed Component**:

```jsx
<div className="relative w-full" style={{ paddingBottom: '56.25%' }}>
  <iframe
    src={`https://player.vimeo.com/video/${videoId}?title=0&byline=0&portrait=0`}
    className="absolute top-0 left-0 w-full h-full"
    frameBorder="0"
    allow="autoplay; fullscreen; picture-in-picture"
    allowFullScreen
  />
</div>
```

**Thumbnail Handling**:

- Fetch Vimeo thumbnail via API: `https://vimeo.com/api/v2/video/${videoId}.json`
- Use custom thumbnail images uploaded to `/public/images/` for better control
- Implement blur-up loading effect for premium feel
